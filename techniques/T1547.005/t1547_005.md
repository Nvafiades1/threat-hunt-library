## Overview

This detection identifies adversaries modifying **LSA-related registry values** that control which Dynamic Link Libraries (DLLs) are loaded by the Local Security Authority Subsystem Service (LSASS) on boot. The query looks for suspicious additions to the following multi-string (REG_MULTI_SZ) values:

* `HKLM\SYSTEM\CurrentControlSet\Control\Lsa\Security Packages`
* `HKLM\SYSTEM\CurrentControlSet\Control\Lsa\Authentication Packages`
* `HKLM\SYSTEM\CurrentControlSet\Control\Lsa\Notification Packages`
* `HKLM\SYSTEM\CurrentControlSet\Control\Lsa\Providers`

An attacker who inserts a malicious DLL name into any of these lists gains **SYSTEM-level persistence** and direct access to credential material once LSASS loads the DLL.

The logic relies on **CrowdStrike FDR** `AsepValueUpdate` events (ASEP = ‚ÄúAuto-Start Extension Point‚Äù) which capture registry value changes. It parses the null-delimited string stored in `RegStringValue`, expands each DLL entry, and flags any DLL **not in a hardened baseline of legitimate Microsoft authentication packages**.

## Security Implications (‚Äúwhy is this bad?‚Äù)

**Persistent SYSTEM Code Execution** ‚Äì DLLs listed here load every boot inside LSASS.  
**Credential Theft** ‚Äì Code inside LSASS can read hashes, tickets, and plain-text passwords.  
**Privilege Escalation** ‚Äì LSASS runs as SYSTEM; attackers elevate automatically.  
**Defense Evasion** ‚Äì Living in a trusted process and abusing a legitimate mechanism.  
**Potential C2 / Exfil** ‚Äì Malicious SSPs can exfiltrate credentials or open network sockets from LSASS.  

## Detection Specifics

### Data Source

| Event Stream | Filter | Key Fields |
| :----------- | :----- | :--------- |
| **CrowdStrike FDR ‚Äì `AsepValueUpdate`** | Registry value update | `TimeGenerated`, `ComputerName`, `ContextProcessId`, `RegValueName`, `RegObjectName`, `RegKeyName`, `RegStringValue` |
| **CrowdStrike FDR ‚Äì `ProcessRollup2`** | Standard telemetry | `TimeGenerated` (alias `PR_Time`), `ComputerName`, `TargetProcessId`, `ImageFileName`, `CommandLine` |

### Detection Pattern (pseudo-logic)

  **Select Registry Change Events**  
   * Source: CrowdStrike FDR `AsepValueUpdate`.  
   * Conditions:  
     - `RegValueName` IN {`Security Packages`, `Authentication Packages`, `Notification Packages`, `Providers`}.  
     - `TimeGenerated` within the look-back window (default **24 h**).  

   **Normalise DLL List**  
   * `RegStringValue` is NULL-delimited.  
   * Trim whitespace ‚Üí replace `\u0000` with `:` ‚Üí store in `RegStringPretty`.  
   * **Discard** empty entries.  

   **Correlate With Writer Process**  
   * **Inner-join** on **`ComputerName` + `PID`**:  
     - Left: `ContextProcessId` from `AsepValueUpdate`.  
     - Right: `TargetProcessId` from `ProcessRollup2`.  
   * Retain `ImageFileName` & `CommandLine` of the writer.

   **(Optional) Baseline Filtering**  
   * Split `RegStringPretty` ‚Üí expand to individual DLL names.  
   * Extract simple file name (`tolower(split(dll,'\\')[-1])`).  
   * Alert only when DLL **NOT IN** hardened baseline list (e.g. _msv1_0.dll_, _kerberos.dll_, ‚Ä¶).  

   **Output / Alert Fields**  
   | Field | Purpose |
   | :---- | :------ |
   | `TimeGenerated` | When registry was modified |
   | `ComputerName` | Hostname |
   | `ImageFileName` | Executable that wrote the value |
   | `CommandLine` | Full command line |
   | `RegValueName` | LSA package touched |
   | `RegKeyName` / `RegObjectName` | Registry key paths |
   | `RegStringPretty` | DLL list (`:`-delimited) |



## Existing Controls

### LSA Protection ‚Äì **RunAsPPL (Protected Process Light)**

| Aspect | Detail |
| :----- | :----- |
| **Purpose** | Prevents unsigned or lower-trust code from loading into LSASS. |
| **How It Works** | Setting `HKLM\SYSTEM\CurrentControlSet\Control\Lsa\RunAsPPL = 1` causes **Smss.exe** to spawn LSASS as a *WinTcb-Light* protected process.<br>Kernel Mode Code Integrity (KMCI) validates every DLL before mapping; only Microsoft-signed or WHQL-signed binaries with the correct EKU are allowed.<br>If validation fails, the load returns **STATUS_INVALID_IMAGE_HASH** and an ETW 3065/3086 event. |
| **How to Verify It Is Enabled** | **Registry**: `RunAsPPL` (DWORD) = 1.<br>**BCDEdit**: `bcdedit /enum {current}` ‚Üí look for `launchsecurityservices Yes`.<br>**Process Explorer**: LSASS shows **Protection: Protected**.<br>**PowerShell**: `(Get-Process lsass).ProtectionLevel` ‚Üí **PsProtectedSignerWinTcb**. |
| **When It Works Correctly** | Secure Boot and HVCI are enabled; host has rebooted since registry change; OS ‚â• Windows 8.1. |
| **Limitations** | Malicious but trusted-signed DLLs still load.<br>Vulnerable drivers can disable KMCI before LSASS starts.<br>**Credential Guard does *not* block SSP loads**‚Äîit only isolates secrets; a signed rogue SSP still executes inside LSASS. |



### Application Control ‚Äì **WDAC / AppLocker / Third-Party Allow-listing**

| Aspect | Detail |
| :----- | :----- |
| **Goal** | Block unsigned or unapproved DLLs **before** they execute. |
| **Mechanism (WDAC)** | Deploy WDAC in **Audit ‚Üí Enforce**.<br>Supplemental policy `LSA_SSP_Allow.xml` lists approved SSP hashes; enable **Options 13 & 16**. |
| **How to Verify It Is Blocking SSP Loads** | **Event Viewer** ‚Üí *Applications and Services Logs ‚Üí Microsoft ‚Üí Windows ‚Üí CodeIntegrity ‚Üí Operational*:<br>‚Ä¢ **Event 3076**: DLL allowed.<br>‚Ä¢ **Event 3077/3078**: DLL blocked; shows rule ID.<br>**WDAC policy viewer** confirms policy ID matches. |
| **AppLocker (legacy)** | **Event 8004** in *Microsoft-Windows-AppLocker/Packaged-app-Deployment* signals DLL blocked by DLL rule. |
| **Third-Party** | EDRs expose their own telemetry (e.g., Carbon Black policy block event). |



### Credential Guard ‚Äì **VBS Secret Isolation (does *not* block SSPs)**

| Aspect | Detail |
| :----- | :----- |
| **Purpose** | Moves hashes/TGTs out of LSASS into **LSAISO.exe** running in Virtual Secure Mode (VSM) to weaken post-exploitation credential theft. |
| **How It Works** | With VBS enabled and `LsaCfgFlags` = 0x1 or 0x2, LSASS brokers credential calls through a secure channel to the isolated process. |
| **Relationship to SSP Loads** | Credential Guard **does not** inspect or block DLLs entering LSASS.<br>A signed malicious SSP that clears RunAsPPL checks will still load‚Äîeven with CG active.<br>The rogue SSP cannot read protected memory but can intercept plaintext creds via normal LSA callbacks. |
| **How to Block SSPs with CG (Win 11 24H2+)** | Set `LsaCfgFlags = 0x80000000`.<br>Windows refuses to load any third-party SSP/AP package (even Microsoft-signed but out-of-box).ETW 3086 ‚Üí **STATUS_INVALID_IMAGE_HASH**; DeviceGuard **Event 7001** logged. |
| **Limitations** | The 0x80000000 flag exists only on Windows 11 24H2+; earlier OSes rely on RunAsPPL + WDAC. |



## Verification & Troubleshooting Checks

| Control | Quick Check | Evidence of Block |
| :------ | :---------- | :--------------- |
| **RunAsPPL** | `reg query HKLM\...\RunAsPPL` == 0x1 | ETW 3065/3086 if load blocked; `Get-WinEvent -Logname Microsoft-Windows-Kernel-Process` |
| **Credential Guard** | `Get-ComputerInfo | Select DeviceGuard*` ‚Üí `CredentialGuardRunning : True` | Event 4218 in **Microsoft-Windows-DeviceGuard/Operational** shows DLL blocked due to CG policy |
| **Group Policy / RSOP** | `gpresult /r /scope computer` or `rsop.msc` ‚Üí *Computer Config ‚Üí Admin Templates ‚Üí System ‚Üí Device Guard ‚Üí Turn On VBS* | Policy shows ‚ÄúCredential Guard with UEFI lock‚Äù; blocking reflected in DeviceGuard events |
| **WDAC** | `Get-CimInstance -ClassName Win32_TrustStatus` or `auditpol /get /subcategory:"Other System Events"` | CodeIntegrity 3077/3078 entries with file path |



## Known Gaps

1. **DLL Signed but Malicious** ‚Äì Requires reputation scoring or memory-scan follow-up.  
2. **Suppressed Telemetry** ‚Äì Host offline or sensor disabled; registry changes may be missed.
